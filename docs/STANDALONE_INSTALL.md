# 오프라인 · 베타 번들 단독 실행

## 동작 방식

**별도 백엔드 없이** 빌드·실행하면 **오프라인·베타 번들**(로컬 데이터)이 기본입니다.

- **게시물**: 이 PC·기기 로컬만 사용 (다른 사용자와 공유되지 않음).
- **상점·가방**: 베타용 별조각·상품; 환경에 따라 **앱 재설치·삭제 시 초기화**될 수 있음.
- **채팅**: 서버 실시간 대신 **베타·로컬 채팅** 화면.
- **이벤트·출석**: 앱에 포함된 베타 공지·로컬 저장 데이터.
- **타로·오라클·캡처**: 기존처럼 `assets/www_gggom/` 로 오프라인 동작.

## Windows 실행 파일 만들기

```powershell
cd 프로젝트\gggom0505_0403
flutter build windows --release
```

결과: `build\windows\x64\runner\Release\` 폴더 전체를 복사해 두면, **다른 PC에도** (같은 아키텍처에서) 비슷하게 단독 실행할 수 있습니다.  
설치 프로그램(MSIX·Inno Setup 등)은 필요 시 별도로 만드시면 됩니다.
