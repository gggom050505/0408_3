2. `품목 진열하기` 클릭.
3. 드롭다운에서 아래가 가능한지 확인:
   - 카탈로그 전체 품목 선택 가능
   - 타입 라벨(`[카드 덱]`, `[이모티콘]` 등) 표시

4. 임의 품목 1개 등록 후 `둘러보기`에서 진열 확인.

관련 파일:
- `lib/widgets/personal_shop_screen.dart`
- `lib/standalone/local_peer_shop_repository.dart`

2. 인트로/완료 차단 화면에서 하단 버튼이 잘리지 않고 스크롤로 접근 가능한지 확인.
3. 카드 뒤집기 후 큰 보기에서 카드 이미지 전체가 화면 안에 보이는지 확인.

관련 파일:

2. 네트워크 실패 상황에서 이미지 `asset` 폴백이 동작하는지 확인.
3. 광고 영상 재생 시도 순서 확인:

## 2) 최근 변경 핵심 요약 (이번 작업분)

### A. 상점 가격 정책 조정

- 이모티콘:
  - 유니크 12종(`emo_asset_01`~`12`) = `1~3`
  - 일반 = `1~4`
  - 파일: `lib/config/bundle_emoticon_catalog.dart`
- 슬롯:
  - 기본 슬롯 제외 = `2~5`
  - 파일: `lib/data/slot_shop_assets.dart`
- 카드 뒷면:
  - `3~6`
  - 파일: `lib/data/card_back_shop_assets.dart`
- 오라클:
  - 유니크 24종(`oracle-card-01`~`24`) = `5~7` 유지
  - 일반(`25`~`80`) = `4~9`
  - 파일: `lib/data/oracle_assets.dart`
- 한국전통 메이저:
  - 유니크 7종(`korea-major-00`~`06`) = `6~8` 유지
  - 일반(`07`~`21`) = `5~10`
  - 파일: `lib/config/korea_major_card_catalog.dart`
- 로컬 상점 가격 정규화도 동일 반영:
  - `lib/standalone/local_shop_repository.dart`

### B. 첫 가입/첫 설정 지급 변경

- 첫 설정 완료 시 지급:
  - 별조각 `20`
  - 이모티콘 `7`
  - 오라클 `8`
- 별조각은 **프로필 최초 생성 시 0**, 첫 설정 확인 시 20 보정
- 첫 설정 완료 후 자동으로 가방 탭 이동
- 파일:
  - `lib/config/starter_gifts.dart`
  - `lib/models/shop_models.dart`
  - `lib/widgets/first_setup_wizard_screen.dart`
  - `lib/widgets/home_screen.dart`

### C. 이벤트/상점 기능 삭제

- 삭제 요청 반영:
  - 이벤트의 **매일 바뀌는 상점 시세** 제거
  - 이벤트의 **상점 깜짝 선물** 제거
  - 상점의 깜짝 선물 배너/수령 플로우 제거
- 관련 모델/동기화 파일 제거:
  - 삭제됨: `lib/models/surprise_gift_models.dart`
  - 삭제됨: `lib/standalone/surprise_gift_sync.dart`
- 인터페이스 정리:
  - `ShopDataSource`에서 surprise gift 메서드 제거
  - `home_screen.dart`, `shop_tab.dart`, 각 repository 구현 정리

### 9-9) 이벤트 보완 — 재현·체크리스트 (기획 반영용)

아래는 **이벤트 탭·보상·기간**을 손볼 때 문서만 보고도 같은 기준으로 검증할 수 있게 정리한 항목입니다. (코드 변경 전·후 공통으로 사용)

- 관련 위치(참고):
  - UI: `lib/widgets/event_tab.dart`
  - 데이터: `lib/standalone/local_event_repository.dart`
  - 번들 가이드: `lib/config/bundled_event_guides.dart`

- 체크리스트:
  1. **기간 표기**
     - 각 이벤트에 시작·종료·KST 기준이 카드/상세에 보이는가.
  2. **종료 처리**
     - 종료 후 목록에서 숨김 또는 `종료` 표시가 일관되는가.
  3. **중복 수령**
     - 동일 계정으로 보상을 두 번 눌러도 한 번만 지급되는가(또는 “이미 수령” 안내).
  4. **idempotent**
     - 앱 재시작·네트워크 끊김 후에도 수령 상태가 깨지지 않는가.
  5. **실패 UX**
     - 지급 실패 시 재시도 가능한가, 성공 시 스낵바/상태 갱신이 맞는가.
  6. **운영 플래그**
     - 테스트 이벤트만 켜는 define 이 있으면 프로덕션 빌드에서 꺼져 있는가.
  7. **오프라인 번들**
     - `flutter run --dart-define=GGGOM_OFFLINE_BUNDLE=true` 로 이벤트 목록이 정상인가.
